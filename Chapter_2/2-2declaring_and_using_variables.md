## 変数宣言の構文
- const 変数名 = 式;
- 変数は、値に好きな名前をつけて保存しておけるもの
- 代入とは値が変数に入ること
- 複数の変数宣言を,をつなげることが可能 
- しかし,を用いた書き方はメンテナンスが劣るため余り使われいない

## 識別子
- 編成宣言で使える名前は「識別詞」と呼ばれる。
- どんな文字列でも名前に使えるわけではなく、一定のルールに従う必要がある。

### 識別子の特徴
- 文字列がルールを満たしていても、予約語は識別子として使えない。
- 予約語とは言語仕様に組み込まれた特別な単語であり、例えば`if`や`const`がこれに当たる。
- これら予約語は編成名として使用できない。

### 識別子に使える文字
- 半角アルファベット
- ひらがな
- カタカナ
- 漢字
## 識別子に利用可能な文字

- Unicode（ユニコード）という文字コード規格で定義されている文字が識別に利用される。
- 記号系の多くの文字は識別子には利用できない。
- **例外的に使用可能な記号**
  - ドル記号 (`$`)
  - アンダースコア (`_`)

## 識別子に使用できない文字

- 空白文字
- 記号類（基本的に）

## 先頭に利用可能な文字と制限

- 先頭文字には数字は利用できない
- ただし、数字は**先頭以外の位置**でのみ識別子に利用可能

### 例

| 識別子 | 妥当か？ | 理由 |
|-|-|-|
| foo1 | 〇 | 先頭は大文字・小文字のアルファベットや許可された記号、数字は2文字目以降なのでOK |
| 1foo | × | 先頭が数字なので識別子として不適切 |

### アルファベットの使用

- アルファベットの大文字・小文字は自由に使える
- プログラミング言語によっては慣習や規則があるが、TypeScript自体に強制的な制限はない

### TypeScriptにおける命名の慣習

- 変数名は小文字で始めることが多い
- 型名は大文字で始めることが多い
- これはあくまで慣習であり、言語仕様としての制限はない

## 変数に型注釈を与える
### 型注釈とは

- **型注釈（Type Annotation）**  
  変数を宣言するときに、その変数がどの型の値をもつかを明示的に指定するものです。

- 変数宣言の構文例  
  ```typescript
  let 変数名: 型;
  ```
  この中の「:`型`」の部分が型注釈です。

---

### 変数の型とは？

- 型は変数にどのような値が入り得るかを示します。
- 例えば、`string` 型は「文字列」を表します。

```typescript
let プリーティング: string;
let ターゲット: string;
```

この例では、`プリーティング`や`ターゲット`には「文字列のみ」を代入可能という意味になります。

---

### 型チェックとコンパイルエラー

- 型注釈に基づき、TypeScriptのコンパイラは型の不一致を検知します。
- 型の不一致はコンパイルエラーとなり、バグの混入を防ぎます。
- この厳密な型チェックは、TypeScriptを用いる際に非常に頻繁に体験することになります。

---

### 型チェックの詳細

- 代入などの式は常に何らかの型を持つ。
- 型のチェックは常に「型同士」で行われる。
- 式の型と代入先の変数の型が一致しない場合、エラーとなる。

---

### まとめ

- TypeScriptの型注釈は、コードの安全性と信頼性を高める重要な機能。
- 変数宣言時に型を明示することで、想定外の値の代入を防止。
- コンパイル時に型エラーを検出し、開発効率と品質向上に寄与する。

## letによる変数宣言と変数への再代入
TypeScriptには`const`での変数宣言の他に、`let`を使った変数宣言方法があります。`let`を使うと、再代入可能な変数を宣言できます。

#### letの特徴
- 再代入が可能
- 宣言時に初期値を与えなくてもよい（ただし、使用前に必ず初期化されている必要がある）
- スコープはブロックスコープ（`{}`内に限られる）

---

### 例：letでの変数宣言と再代入

```typescript
let greeting = "Hello";
console.log(greeting); // Hello

// 変数greetingに再代入
greeting = greeting + " World";
console.log(greeting); // Hello World
```

このサンプルでは、最初に`greeting`に文字列`"Hello"`を代入しています。その後、`greeting`に別の文字列を足して再代入しています。

---

### constとの違い

`const`で宣言した変数は再代入ができません。`const`変数に再代入しようとすると、コンパイルエラーが発生します。

```typescript
const greeting = "Hello";
greeting = "Hello World"; // エラー: Cannot assign to 'greeting' because it is a constant.
```

---

### 初期値を与えずにletで宣言する例

```typescript
let count: number;

count = 10;
console.log(count); // 10

count = 20;
console.log(count); // 20
```

`let`変数は宣言時に初期値を与えなくてもよいですが、使用する前には必ず初期化しなければなりません。TypeScriptのコンパイラは初期化していない変数を使用すると、エラーを出します。

---

### まとめ
- `let`は再代入が可能な変数宣言方法
- `const`は再代入不可で、定数に適している
- `let`なら宣言時に初期化しなくてもよいが、使用前に初期化が必須
- 型注釈を付けて変数宣言も可能（例：`let count: number;`）

TypeScriptでは、変数宣言に主に2つの方法があります。`const` と `let` です。  
これらは変数の再代入の可否と初期化方法に違いがあります。

---

#### `const` の特徴

- 宣言時に必ず初期値を代入しなければならない。
- 一度値を代入すると、その後の再代入はできない（定数のように振る舞う）。
- 再代入不可なので、意図せぬ値の書き換えミスを防止できる。
- 一般的なWebアプリケーション開発では、可能な限り`const`を使うことが推奨される。

```typescript
const pi = 3.14;
// pi = 3.1415; // エラー: 再代入は許されない
```

---

#### `let` の特徴

- 宣言時に初期化を省略できる。
- 後から値を再代入できる。
- ただし、ほとんどの変数は一度代入すれば十分なため、`let`を使う機会はそれほど多くない。
- `let`を多用するとプログラムの可読性が下がり、他者がコードを理解する際に負担がかかる。

```typescript
let counter;
counter = 0;
counter = counter + 1;
```

---

#### なぜ`const`を優先すべきか？

- `const`を使うことで「後から値が変更されない」という意図が明示され、コードが読みやすくなる。
- 複数人での開発や大規模なメンテナンスでは、可読性と予測可能な挙動が重要。
- `let`を使用する場合は、本当に後で値を変更する必要があるケースに限定すべき。

---

#### まとめ

| キーワード | 初期化        | 再代入      | 使用推奨度              |
|-------------|---------------|-------------|-------------------------|
| `const`     | 必須          | 不可        | 基本的にこちらを優先    |
| `let`       | 任意（省略可）| 可          | 再代入が必要な場合のみ  |

---

### 補足：`const`は定数か？

- `const`キーワードは他言語では「定数」を宣言する意味で使われることもあるが、
- TypeScriptにおいては「再代入できない変数」として扱うのが一般的な見方。
- オブジェクトの中身は変更可能なので、イミュータブルを保証するものではない点に注意。

```typescript
const obj = { name: "TypeScript" };
obj.name = "TS";  // OK（オブジェクトの内容は変更可能）
// obj = { name: "JavaScript" }; // エラー: 再代入は不可
```
